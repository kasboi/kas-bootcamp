## Form and Input Tags

We all have had to fill forms at one point, either physically or online. Forms are used to gather information such as academic bio-data, user registration or login etc.

There are dedication tags in HTML for collecting form data

`<form>`: Used to create a form that collects user input and submits it to a server for processing. It typically includes input fields and a submit button.

`<input>`: Used to create various input fields within a form, such as text boxes, radio buttons, checkboxes, etc. The type attribute defines the kind of input (e.g., text, email, password, checkbox, etc.).

Create a new HTML file named `survey.html` in the `html_basics` folder. The folder structure should look like this:

```
[html_basics]
|
|--[img]
|
|--chapter.html
|
|--survey.html
|
|--index.html
```

### Form element

The `form` element is the appropriate container for all form inputs.

```html
<body>
    <div>
        <h1 id="title">CareerConnects Survey Form</h1>
        <p id="description">
            Thank you for taking the time to help us improve the plaform
        </p>
    </div>
    <form id="survey-form"></form>
</body>
```

### Form inputs

Form inputs are used to specify the type of data to be collected ranging from text, email, password, colour, checkbox etc.

```html title="/html_basics/survey.html" {10-19}
<body>
    <div>
        <h1 id="title">CareerConnects Survey Form</h1>
        <p id="description">
            Thank you for taking the time to help us improve the plaform
        </p>
    </div>
    <form id="survey-form">
        <!-- Name Input -->
        <div class="form-class">
            <label for="name">Name</label>
            <input
                type="text"
                id="name"
                class="form_input"
                placeholder="Enter your name"
                required
            />
        </div>
    </form>
</body>
```

**Breakdown:**

1. `<div class="form-class">` serves as a container for the input and label elements
2. `<label for="name">` is used to provide a relationship between a text and an input. The `name` attribute links to the `id` of the input
3. `<input type="text" />` - The text input is used for gathering plain text data.

-   `id="name"` - Unique identifier for the input which allows the label to know which element to focus on when clicked
-   `class="form_input"` - This class name will be applied to multiple inputs which will use the same styling later on in the course.
-   `placeholder="Enter your name"` - The placeholder is a text shown in the input which is cleared immediatelly the use enters data into it
-   `required` - This tells the browser that the input field field is important and must be filled before submission

```html title="/html_basics/survey.html" showLineNumbers {14-23}
<form id="survey-form">
    <!-- Name Input -->
    <div class="form-class">
        <label for="name">Name</label>
        <input
            type="text"
            id="name"
            class="form_input"
            placeholder="Enter your name"
            required
        />
    </div>
    <!-- Email Input -->
    <div class="form-class">
        <label for="email">E-mail</label>
        <input
            type="email"
            id="email"
            class="form_input"
            placeholder="Enter your email"
            required
        />
    </div>
</form>
```

**Breakdown:**

1. The email input accepts valid email address and throws an error if the pattern expected does not match

### Dropdown

```html title="/html_basics/survey.html" showLineNumbers {19-25}
<form id="survey-form">
    <!-- Some code above -->
    <!-- Age [minimum of 18 and maximum of 75] -->
    <div class="form-class">
        <label for="age">Age</label>
        <input
            type="number"
            id="age"
            class="form_input"
            placeholder="Input your age"
            min="18"
            max="75"
            required
        />
    </div>
    <!-- Dropdwon menu -->
    <div class="form-class">
        <p>Which option best describes your current role?</p>
        <select name="dropdwon" id="dropdown" class="form_input">
            <!-- The first option should have an emtpy value for the required attribute to work -->
            <option value="" disabled selected>Select your role</option>
            <option value="student">Student</option>
            <option value="full-time">Full-time</option>
            <option value="part-time">Part-time</option>
            <option value="others">Others</option>
        </select>
    </div>
</form>
```

**Breakdown:**

1. The number input tag only allows numeric values
2. `min` and `max` attributes are used to specify the minimum and maximum number values allowed
3. The dropdown menu is special because it uses a unique tag `select` and `option`

-   `<select>` is the container. The `name` attribute is used when sending the form data to the server to identify the input and its value
-   `<option>` tags are used to list the values within the dropdown
-   `value` attribute is the value sent to the server along with the name of the input
-   `disabled` attribute makes it so that the option cannot be selected by the user
-   `selected` is used to control the default value selected within the dropdown, the user can change this

### Radio buttons

```html
<!-- Radio Buttons -->
<div class="form-class">
    <p>Would you recommend CareerConnects to a friend?</p>
    <input type="radio" value="definitely" id="definitely" class="form_radio" />
    <label for="definitely">Definitely</label>
    <!-- Break line -->
    <br />
    <input type="radio" value="maybe" id="maybe" class="form_radio" />
    <label for="maybe">Maybe</label>
    <!-- Break line -->
    <br />
    <input type="radio" value="nah" id="nah" class="form_radio" />
    <label for="nah">Nah</label>
</div>
```

**Breakdown:**

1. `<input type="radio" />` - Used to provided limited number of options from which the user can pick only one.
2. Requires a `name` attribute to link related buttons
3. `<br />` to split buttons into new lines


### Checkbox

Checkboxes are used to provide limited values to the user, from which they can select onr or multiple

```html
<!-- Checkbox -->
<div class="form-class">
    <p>What would you like to see improved? (Check all that apply?)</p>
    <input
        type="checkbox"
        id="front"
        class="form_check"
        name="front"
        value="front"
    />
    <label for="front">Front-end Projects</label>
    <input
        type="checkbox"
        id="back"
        class="form_check"
        name="back"
        value="back"
    />
    <label for="back">Back-end Projects</label>
    <input
        type="checkbox"
        id="data_science"
        class="form_check"
        name="data_science"
        value="data_science"
    />
    <label for="data_science">Data Science</label>
    <input
        type="checkbox"
        id="project"
        class="form_check"
        name="project"
        value="project"
    />
    <label for="project">Project Management</label>
</div>
```

**Breakdown:**

1. `<input type="checkbox" />` for declaring checkbox input
2. `name` and `value` attributes are sent to the server for every option the user selects

### Textarea

Alows you to accept long form of plain text input from the user. It is a flexible input type because it allows the user to manually resize the input box

```html
<!-- Textarea input -->
<div class="form-class">
    <textarea
        name="text"
        id="textarea"
        cols="30"
        rows="6"
        class="form_input"
        placeholder="Enter your comments here"
    ></textarea>
</div>
```

**Breakdown:**

1. `<textarea>` - The text area input uses a dedicated tag.
2. `cols` attribute specifies the initial width of the element
3. `rows` attribute specifies the initial height of the element

### Submit & Reset

`<submit>` is used to send all inputs collated in the form to the server or perform an operation

`<reset>` is used to reset all the input data within the form

```html
<!-- Submit & Reset buttons -->
<div class="form-class">
    <input type="submit" value="Submit" class="form_submit" />
    <input type="reset" value="Reset" class="form_reset" />
</div>
```
