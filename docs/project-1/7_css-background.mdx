## `background`

Let's talk about CSS `background` property before moving on with our mini project. The `background` property is a shorthand for many background properties.

## `background-color`

This we have used multiple times for defining the background colour of an element.

```css
p {
  background-color: #f3f;
}

section {
  /* This will also work */
  background: rebeccapurple;
}
```

## `background-image`

In the example below we have a container which has a width of `1200px` and height of `600px`. The background image applied to it has a width of `556px` and height of `371px` and you can see it being repeated to fill the entire `div` container.

## `background-repeat`

This property is used to specify how the image should be repeated, by default it is repeated vertically and horizontally. You can change this to:

- `no-repeat`: The image does not repeat in either axis
- `repeat-x`: To repeat in the vertical direction only
- `repeat-y`: To repeat in the horizontal direction only

## `background-size`

The background-size property in CSS allows you to control how a background image is resized to fit within an element's dimensions. Two commonly used values for this property are contain and cover, each of which behaves differently:

- `background-size: contain;`

  **Description**: Scales the background image to fit within the element while preserving its aspect ratio. This means that the entire image will be visible, but there may be empty space (often a "letterbox" effect) if the image and element have different aspect ratios.

  **Use Case**: Best when you want the entire image to be visible without any cropping, such as for logos, icons, or images that need to be fully seen within a designated area.

- `background-size: cover;`

  **Description**: Scales the background image to completely cover the element's background area, while preserving the image's aspect ratio. This can result in part of the image being cropped if its aspect ratio does not match that of the element.
  **Use Case**: Useful for full-screen backgrounds, banners, and other situations where you want the background to fully cover the element without worrying about empty spaces, even if parts of the image may be cropped.

### Summary Table

| Property             | `contain`                                            | `cover`                                          |
| -------------------- | ---------------------------------------------------- | ------------------------------------------------ |
| **Resizes Image To** | Fit within the element                               | Cover the entire element                         |
| **Aspect Ratio**     | Maintained                                           | Maintained                                       |
| **Empty Space**      | Yes, if aspect ratio differs from the element        | No                                               |
| **Cropping**         | No cropping                                          | Cropped if aspect ratio differs from the element |
| **Best For**         | Icons, logos, or any content needing full visibility | Full backgrounds, banners, hero images           |

Choosing between `contain` and `cover` depends on whether you prioritize full image visibility (`contain`) or a fully filled background area (`cover`).

You can also use other units to specify the image size such px and percentage.

```css
div {
  background-size: 30%;
}

section {
  /* width: 200px & height: 100px */
  background-size: 200px 100px;
}
```
